## 🤝 Learning to Work in a Team

The biggest shift in my growth as an engineer wasn't a technology. It was learning how to **work with other people on the same codebase**.

Here's what that actually looked like:

**Git became a communication tool, not just a save button.**
I learned that commits tell a story. A message like `fix bug` means nothing. A message like `fix: prevent duplicate email on CSV import` means everything. Your teammates — and your future self — will thank you.

**Branches protect everyone.**
Working on the same `main` branch as a team is a recipe for chaos. Feature branches let everyone work without stepping on each other. Pull requests let the team review before anything merges.

**Breaking things is fine. Breaking shared code is not.**
When you work solo, you can afford to be messy. When you work in a team, your mess becomes someone else's problem. That pressure made me write cleaner, more careful code.

**Communication > Code.**
The best pull request in the world means nothing if your teammate doesn't know what problem you were solving. I learned to write clear descriptions, ask questions early, and not disappear when things got stuck.

**You learn faster in a team.**
When someone on your team does something you've never seen before, you get to ask "how did you do that?" That kind of learning doesn't happen alone.
